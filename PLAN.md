## 1. 项目说明
1. 个人任务日程管理网站，它可以创建任务，任务可以添加子任务，它可以在安卓手机、苹果手机、微软电脑、苹果电脑、linux电脑上即开即用，符合各端的屏幕尺寸
2. 主页面应该包含三个部分，可以通过点击切换：需求、迭代、看板
    2.1. 需求页面：展示所有的任务，具体数据以页面大小为准，数据展示的优先级排序为'标题、状态、优先级、迭代、预估工时、完成工时'，可以配置显示任务属性中的哪些数据。任务及子任务按照树状展示，默认不展开树。支持按照优先级、状态、迭代等进行筛选和排序，支持搜索功能，支持批量操作（如批量修改状态、批量删除等），支持导出功能（如导出为Excel、CSV等格式）。主页面要可以修改展示的任务的数据。点击任务后可以进入任务详情页面，展示该任务的详细信息和子任务列表，支持编辑任务属性、添加子任务、修改子任务状态等操作。
    2.2 迭代页面：进入时展示当前处于执行中的迭代，展示该迭代的详细信息和处于该迭代的任务列表，支持编辑迭代属性、添加任务到迭代、修改任务状态等操作。迭代按照开始时间排序，迭代卡片展示迭代名称、开始时间、结束时间、预计工时、完成工时，剩余工时（通过包含的所有任务计算出来）包含的任务数量。点击任务可以进入任务详情页面。
    2.3 看板页面：分为多个方块，每个方块代表不同的统计方法。首先实现迭代统计方块，通过折线图展示：按照最新的迭代，x轴为迭代起止时间，y轴为每一天的剩余工时和完成工时。其次是阶段性统计方块：统计一段时间时间内的工时投入，显示你在这段时间内投入了多少时间，默认按照月进行统计。但是可以调整时间范围为周、季度、年和指定起止时间。

## 2. 数据说明
1. 任务属性包含：ID（不显示，不允许修改，服务器生成）	标题	状态	优先级	父需求(需要为父需求的ID）	迭代 预估工时	完成工时 预计开始	预计结束	创建时间	完成时间	剩余工时	超出工时	进度，示例在example中
2. 状态包含：未开始、进行中、已完成、取消。状态间可以流转，并且记录到达每个状态的最新时间戳。未开始只能进入进行中和已取消
3. 优先级包含：低、中、高。页面显示时分别使用灰色，蓝色和红色。
4. 迭代包含：迭代包含名称数据和预计开始时间和结束时间，以及状态（规划中，执行中，结束），以及文本数据-迭代总结。迭代应该拥有系统生产的不显示的默认ID，可以配置迭代的开始时间和结束时间。同一时间只允许一个迭代处于执行中，迭代的状态只允许手动调整。
5. 任务可以添加子任务，子任务的父需求为父任务的ID。父需求的预估工时为所有子任务预估工时之和，父需求的完成工时为所有子任务完成工时之和，父需求的剩余工时为所有子任务剩余工时之和，父需求的超出工时为所有子任务超出工时之和，父需求的进度根据所有子任务的进度计算得出。父需求的状态独立。
6. 任务的进度根据完成工时和预估工时计算得出
7. 任务的剩余工时根据预估工时和完成工时计算
8. 任务的超出工时根据完成工时和预估工时计算得出

## 技术选型
> 你可以自行配置环境，安装依赖，但是要保证可移植性。也就是在任何环境中都可以安装依赖并且运行。以下是推荐的技术选型：
1. 基础工具
Node.js（v18+ 推荐）
pnpm / yarn / npm（推荐 pnpm）
Vite（更快的构建性能）
2. 框架
Vue
3. UI 方案
TailwindCSS + 动效库（Framer Motion / VueUse Motion）
4. UI / UX
响应式布局：Flexbox / CSS Grid / Tailwind utility 类
动画支持：
Vue：VueUse Motion

支持触摸手势、列表拖拽等

性能优化：
可以选择使用 Service Worker 提升离线性能

资源懒加载、分包加载

避免过度动画影响低配设备体验
5. 数据存储
本地存储可以使用数据库，但是要支持webdav定时备份数据库。这样最大化性能和安全性。
